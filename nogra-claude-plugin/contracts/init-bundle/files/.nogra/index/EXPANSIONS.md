Add Nogra surfaces only when repeated work proves they are needed. Do not add
empty folders or parallel records for appearance.

## Add When Needed

| Surface | Add when | Why |
|---|---|---|
| Project hub | One folder manages several real projects | Keeps each project state-local while the hub indexes them |
| Scenario pack | The same drift/risk pattern must be graded repeatedly | Turns incidents into reusable behavior tests |
| Evidence adapter | Native file/test evidence is not enough for a recurring claim | Keeps verification tied to observed behavior |
| Verifier flow | Evidence is noisy or high-stakes enough to need independent review | Separates execution claims from verification judgment |
| Risk registry row | A new system can be read or written | Makes permissions and evidence expectations explicit |
| Brief template | The same scoped work repeats with stable criteria | Reduces ambiguity without hiding approval |

## Wait Instead

Wait when the need happened once, has no owner, has no evidence contract, or
would create a second source of truth.

## Rule

One new surface needs a route, an owner, an evidence expectation and a cleanup
story. Otherwise keep it inside the current brief, run report or decision entry.
